¿Para qué se usan las clases?

Las clases en Python se usan para crear objetos que agrupan datos y funciones en una estructura organizada. Esto se conoce como programación orientada a objetos (POO). Las clases permiten modelar entidades del mundo real con atributos (datos) y métodos (funciones) que pueden operar sobre esos datos.

Conceptos básicos

  • Clase: Es una plantilla para crear objetos. Define un conjunto de atributos y métodos.
  • Objeto: Es una instancia de una clase. Cada objeto tiene sus propios valores para los atributos definidos en la clase.
  • Atributos: Son variables que pertenecen a la clase o a los objetos.
  • Métodos: Son funciones que pertenecen a la clase y pueden actuar sobre los atributos.

Beneficios de utilizar clases en Python

  • Organización: agrupa los datos y las funciones relacionadas en una estructura legible y coherente.
  • Reutilización: se pueden crear más de un objeto desde la misma clase sin tener que repetir código: uno de los principios fundamentales de la programación: DRY (Don’t Repeat Yourself).
  • Abstracción: oculta los detalles más complejos para tener una interfaz sencilla
  • Modularidad: ayuda a separar las responsabilidades en el código.

¿Qué método se ejecuta automáticamente al crear una instancia?

__init__ es el método que se ejecuta al crear una instancia de una clase, el llamado también “constructor”, cuya función es establecer los atributos del objeto.

En el ejemplo final se define el método __init__ que es donde se le pasan los atributos solicitados al Coche (objeto): marca, modelo y año. Siempre se solicita el argumento “self” para hacer referencia al objeto que se está creando en ese momento.

Este constructor es fundamental para inicializar los objetos y asegurarse de que cada instancia u objeto, se inicie en un estado correcto y válido.

Decoradores

Son una herramienta que permite modificar el comportamiento de funciones o métodos sin cambiar el código original. El decorador es una función que recibe otra como argumento, la modifica y devuelve una nueva.

Se utilizan para proteger o privatizar variables:

  • Se utiliza '_' (UN guión bajo) delante de una variable para especificar que se declara "protected" (protegida).
  • Se utiliza '__' (DOS guiones bajos) delante de una variable para especificar que se declara "private" (privada).

Su funcionamiento se aplica con la sintaxis '@nombre_del_decorador' antes de definir la función.

Herencia

Permite crear una nueva clase (subclase) en base a una clase ya existente (clase padre o superclase). La subclase hereda los atributos y métodos de la superclase y de esta manera se promueve y facilita la reutilización del código.

Polimorfismo

Es un principio fundamental de la Programación Orientada a Objetos (POO). Permite facilitar el uso de una interfaz común para diferentes objetos y que el mismo método se utilice pero dando diferentes salidas según se establezca previamente.

El polimorfismo permite que una subclase sobreescriba métodos de la clase padre (superclase).

Ejemplo final

Creamos un coche, mediante la clase “Coche” con los atributos propios (igual que en la vida real) como “marca”, “modelo”, “año”… Y métodos (acciones que tiene el coche) como “arrancar” o “detener”.


            class Coche:
                def __init__(self, marca, modelo, año):
                    self.marca = marca
                    self.modelo = modelo
                    self.año = año

                def arrancar(self):
                    print(f"El {self.marca} {self.modelo} está arrancado.")

                def detener(self):
                    print(f"El {self.marca} {self.modelo} está parado.")

            # Decoradores
            def mayusculas(func):
                def envoltura(self):
                    resultado = func(self)
                    return resultado.upper()
                return envoltura

            class CocheDecorado(Coche):
                @mayusculas
                def arrancar(self):
                    return f"El {self.marca} {self.modelo} está arrancado."

                @mayusculas
                def detener(self):
                    return f"El {self.marca} {self.modelo} está parado."

            # Objetos utilizando DECORADORES (Instancias)
            mi_coche = CocheDecorado("Toyota", "Corolla", 2020)
            tu_coche = CocheDecorado("Honda", "Civic", 2018)

            # Métodos
            print(mi_coche.arrancar())
            print(tu_coche.detener())

            # Herencia
            class Deportivo(Coche):
                def arrancar(self):
                    return f"El {self.marca} {self.modelo} está arrancando con un rugido potente."
                
            class Electrico(Coche):
                def arrancar(self):
                    return f"El {self.marca} {self.modelo} está arrancando silenciosamente."

            # Objetos utilizando HERENCIA y POLIMORFISMO
            deportivo = Deportivo("Ferrari", "340 América", 1951)
            electrico = Electrico("Peugeot", "e-208", 2020)

            # Métodos POLIMÓRFICOS
            arrancar_coche(deportivo)
            arrancar_coche(electrico)
