¿Qué es una API?

API son las siglas de Application Programming Interface. Es un conjunto de definiciones y protocolos que permiten comunicar un software con otro:

  • API de Biblioteca: Es la interfaz que proporciona una biblioteca para que interactúe con funciones y clases. Por ejemplo: la biblioteca 'requests' que Python utiliza para realizar solicitudes HTTP.
    • 
            # Se importa la biblioteca 'requests' para realizar las solicitudes
            import requests

            # Hacer una solicitud GET a una API web
            response = requests.get('https://api.ejemplo.com/datos')

            # Verificar el estado de la respuesta
            if response.status_code == 200: # 200 es el código de estado que devuelve una solicitud HTTP para indicar que se ha realizado correctamente.
                # Procesar los datos de la respuesta (suponiendo que los datos son de tipo JSON)
                datos = response.json()
                print(datos)
            else:
                print(f"Error en la solicitud: {response.status_code}")
  • API Web: Es la interfaz que permite a las aplicaciones web comunicarse con servidores y bases de datos a través de la web; las cuales suelen utilizar HTTP para dichas comunicaciones y formatos JSON o XML para los intercambios de datos.

Características de una API

  • Tiene una interfaz consistente: define funciones y métodos que se deben utilizar de una manera concreta.
  • La API oculta la complejidad del sistema y solo muestra lo necesario para que los desarrolladores interactúen con él.
  • Siguen estándares y convenciones para que los diferentes sistemas se comuniquen correctamente.
  • Suelen tener documentación detallada con información sobre cómo utilizar las diferentes funciones y métodos disponibles.

¿Qué es "PostMan"?

Es una herramienta que utilizan los desarrolladores para probar, desarrollar y documentar las APIs. Ofrece una interfaz gráfica que es muy intuitiva y facilita la creación de solicitudes HTTP y la supervisión de las respuestas que devuelven.

Beneficios de utilizar "PostMan"

  • Su interfaz intuitiva hace que los desarrolladores interactúen con las APIs sin tener que escribir código
  • Soporta varios métodos HTTP y configuraciones de solicitudes.
  • Permiten automatizar pruebas y procesos repetitivos.
  • Las funciones de equipo facilitan la colaboración entre los miembros de este.
  • Facilita la creación de documentación interactiva.
